# Photos — where they go

Upload your photos into **this `images/` folder** using the exact filenames
below (all lowercase, `.jpg`). When a file's name matches, it appears on the
site automatically — no code editing needed. Until a photo is uploaded, that
spot shows a soft tan placeholder (no broken-image icons).

## Filenames

| Filename | Where it appears | Best shape |
|---|---|---|
| `hero.jpg` | Big background photo behind the headline | Wide landscape (e.g. 1920×1080) |
| `spring.jpg` | Meet the Locals — Spring | Portrait (e.g. 800×1000) |
| `cannoli.jpg` | Meet the Locals — Cannoli | Portrait |
| `dobby.jpg` | Meet the Locals — Dobby | Portrait |
| `mia.jpg` | Meet the Locals — Mia | Portrait |
| `rachel.jpg` | About Me — Rachel's portrait | Portrait |
| `property-outdoor.jpg` | The Property — Outdoor Space | Landscape (e.g. 1200×750) |
| `property-indoor.jpg` | The Property — Indoor Space | Landscape |
| `gallery-1.jpg` … `gallery-8.jpg` | Gallery grid (8 photos) | Square (e.g. 800×800) |

## Tips
- **Filename must match exactly** — `spring.jpg`, not `Spring.jpg` or `spring.jpeg`.
  If your file is a `.png`, either rename it to `.jpg` (renaming the extension is
  fine for the browser) or tell me and I'll switch that slot to `.png`.
- **Keep files reasonably small** (~300 KB–1 MB each) so the site loads fast.
  Most phone photos are fine; very large files just load slower.
- **Shapes** above are ideal, but any photo works — it's cropped to fit the frame.

## How to upload on GitHub
1. Open this `images` folder on GitHub.
2. Click **Add file → Upload files**.
3. Drag your photos in (named as above) and **Commit changes**.
4. Give GitHub Pages a minute, then refresh the site.
